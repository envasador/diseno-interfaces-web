# Órbita 5: Diseñar para todos
## Introducción

Has aprendido a diseñar interfaces atractivas, maquetar con precisión, optimizar multimedia y añadir interactividad profesional. Pero todo eso no tiene valor si **no todos pueden usarlo**.

La accesibilidad web es un cambio de mentalidad: pasar de diseñar para "el usuario promedio" (que no existe) a diseñar para **la diversidad humana real**. Más del 15% de la población mundial tiene alguna discapacidad. En España, aproximadamente 4 millones de personas. Si tu web no es accesible, estás excluyendo a millones de usuarios.

Pero la accesibilidad no es solo un tema ético o legal. Es **calidad**: una web accesible es más fácil de usar, mejor posicionada en Google, con código más limpio y mantenible. Beneficia a todos.

Esta órbita te enseñará a **gestionar la accesibilidad** como parte integral de tus proyectos, no como un parche al final. No te daremos listas interminables de código. Te daremos conceptos fundamentales sólidos, criterios de decisión, y referencias a recursos especializados como **accesible.es** donde profundizar cuando necesites implementar componentes específicos.

**Empieza por lo fundamental:**
1. HTML semántico (80% de la accesibilidad)
2. Navegación por teclado funcional
3. Contraste de color suficiente
4. Textos alternativos en imágenes

El resto lo irás dominando con la práctica, proyecto a proyecto.
