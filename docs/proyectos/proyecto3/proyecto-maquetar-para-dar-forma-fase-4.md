# **FASE 4: RESPONSIVE DESIGN Y LAYOUTS COMPLETOS**

**Criterios:** RA2.c, RA2.d, RA2.f  
**Entrega:** 14 de enero

## **OBJETIVOS DE LA FASE**

Adaptar toda tu aplicación para que funcione perfectamente en todos los dispositivos desde mobile hasta desktop. Implementar Container Queries en componentes clave y crear layouts completos para todas las páginas de tu aplicación.

Esta fase se coordina con **DWEC Fase 4** donde implementas el sistema de rutas y navegación entre todas las páginas.

## **TAREAS**

### **1. Adaptación responsive completa de la aplicación**

Adapta TODA tu aplicación para que funcione en mobile, tablet y desktop. Esto incluye todas las páginas y todos los componentes que has creado.

### **2. Container Queries**

Implementa Container Queries en al menos 1 componente para hacerlos independientes del viewport.

### **3. Páginas completas responsive**

Crea layouts completos y responsive para **todas las páginas** de tu aplicación.

### **4. Estrategia responsive consistente**

Elige y aplica consistentemente una estrategia: mobile-first O desktop-first.

Justifica tu elección en la documentación.

### **5. Testing responsive**

Verifica tu aplicación en los siguientes viewports usando Chrome DevTools:
- 320px (mobile pequeño)
- 375px (mobile estándar)
- 768px (tablet)
- 1024px (desktop pequeño)
- 1280px (desktop estándar)

Usa también Firefox Developer Tools para verificar compatibilidad.

### **6. Documentación en DOCUMENTACION.md**

Añade la **Sección 4: Responsive design** a tu archivo `docs/DOCUMENTACION.md`.

Esta sección debe incluir:

**4.1 Breakpoints definidos:** Lista tus breakpoints con justificación de por qué elegiste esos valores.

**4.2 Estrategia responsive:** Explica si usaste mobile-first o desktop-first y por qué. Incluye ejemplo de código.

**4.3 Container Queries:** Documenta en qué componentes implementaste Container Queries y por qué. Incluye código de ejemplo.

**4.4 Adaptaciones principales:** Tabla resumen mostrando cómo se adapta la aplicación en mobile, tablet y desktop.

**4.5 Páginas implementadas:** Lista las páginas responsive creadas con breve descripción.

**4.6 Screenshots comparativos:** Incluye capturas de al menos 3 páginas mostrando mobile (375px), tablet (768px) y desktop (1280px).

## **ENTREGABLES FASE 4**

- Toda la aplicación adaptada a mobile, tablet y desktop
- Container Queries implementadas en mínimo 2 componentes
- Mínimo 3 páginas completas responsive
- Estrategia responsive consistente aplicada en toda la aplicación
- Testing verificado en 5 viewports diferentes
- Sección 4 del `docs/DOCUMENTACION.md` completada con tabla de adaptaciones y capturas comparativas
